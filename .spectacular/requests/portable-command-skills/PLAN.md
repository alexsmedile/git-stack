## Goal

<!-- One sentence. What does this request change? -->
<!-- Compress the request's intent. Aligns with PRD's Vision or Goals — this is a slice, not a restatement. -->

Generate deterministic, host-neutral command specifications into Claude-native
commands and Agent Skill adapters for project or global installation.

## Constraints

<!-- What's fixed before you start? Inherited from PRD/STACK/PRINCIPLES + request-specific limits. -->

- `skills/git-ops/scripts/git-stack.sh` remains the mechanical source of truth;
  generated command surfaces must not duplicate Git checks.
- Claude plugin commands remain native and namespaced; command skills are an
  opt-in adapter for harnesses without native command support.
- All seven current commands are generated by default; users opt out with
  inverted flags such as `--no-release`.
- Project and global installs must be collision-safe, reversible, and usable
  without executing remote lifecycle scripts.
- Command-skill generation runs from the full public repository checkout so the
  catalog can resolve canonical command bodies; skill-only installs retain the
  portable core.

### How it works now

Claude commands live under `adapters/claude/commands/` and are exposed through
the Claude plugin manifest. Other harnesses receive the portable `git-ops`
skill but do not receive equivalent command entry points. The existing
`install-harness.mjs` installs skills and optional agents, while
`install-shortcuts.mjs` only creates Claude standalone command aliases.

### What changes

Add neutral command specs, a deterministic generator, and installer support for
generated command skills under each harness's project/global skill root. Strip
Claude-only frontmatter and rewrite plugin-root script paths during generation.
Add metadata and validation so generated files can be updated or removed
without overwriting unrelated user skills.

### What stays the same

The Git safety implementation, command behavior, Claude plugin manifest, native
agent model policy, and existing short-command alias installer remain intact.
This change does not promise that every harness exposes literal slash-command
syntax; non-Claude outputs are native skills.

- Chose a neutral command catalog index over maintaining separate command lists;
  existing Claude command bodies remain the canonical workflow text while the
  generator emits host-specific Agent Skill adapters.
- Chose all commands enabled by default with `--no-*` exclusions because first
  setup should be complete while remaining easy to narrow.
- Chose committed/generated Claude outputs plus installer-generated local skill
  outputs because marketplace installs cannot be assumed to execute build hooks.
